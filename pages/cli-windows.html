Usare GitHub Copilot da CLI su Windows

Introduzione

Questa guida spiega come installare e usare GitHub Copilot direttamente dal terminale su Windows (PowerShell, Windows Terminal, Git Bash o WSL).

Cosa Imparerai

Al termine di questa guida sarai in grado di utilizzare GitHub Copilot direttamente dal terminale per ricevere suggerimenti su comandi, spiegazioni di errori e assistenza interattiva tramite chat.

0. Prerequisiti

Prima di iniziare, assicurati di avere:

  • Windows 10 o superiore
  • Account GitHub con Copilot attivo
  • Connessione Internet stabile

Verifica Accesso GitHub Copilot

Assicurati che il tuo account GitHub abbia una sottoscrizione attiva di Copilot. Puoi verificarlo nelle impostazioni del tuo account GitHub.

1. Installazione GitHub Copilot CLI

Installare tramite Winget

Apri PowerShell o Windows Terminal ed esegui il seguente comando:

winget install GitHub.Copilot

Questo comando installa il pacchetto Copilot CLI completo sul tuo sistema.

Importante

Al termine dell'installazione, chiudi e riapri il terminale per assicurarti che le variabili d'ambiente siano aggiornate correttamente.

Verifica Installazione

Per verificare che l'installazione sia andata a buon fine, esegui:

copilot

Se vieni reindirizzato nel terminale GitHub Copilot CLI, l'installazione è corretta e puoi procedere.

2. Login ed Accesso a GitHub

Una volta installato, devi autenticare il CLI con il tuo account GitHub.

Procedura di Login

  1. Avvia il comando di login

    All'interno del terminale Copilot, esegui:

    /login
  2. Seleziona il metodo di autenticazione

    Quando richiesto, seleziona:

    • GitHub.com (per login via browser)
  3. Completa l'autenticazione

    Segui le istruzioni per l'inserimento del codice sul web. Ti verrà chiesto di:

    • Aprire il browser
    • Inserire il codice visualizzato nel terminale
    • Accedere con l'account GitHub che ha Copilot attivo

Conferma Login

Una volta completata l'autenticazione, vedrai il messaggio:
● Signed in successfully as xxxxx! You can now use Copilot.

Ora puoi utilizzare GitHub Copilot CLI.

3. Utilizzo dei Comandi Principali

GitHub Copilot CLI offre diversi comandi per assistere il tuo lavoro quotidiano dal terminale.

3.1. Suggerire Comandi

Usa il comando suggest per ricevere suggerimenti su come eseguire un'operazione:

copilot suggest "descrizione del comando"

Esempio Pratico

copilot suggest "mostra i processi che usano la porta 8080"

Copilot ti fornirà il comando appropriato per il tuo sistema operativo.

3.2. Spiegare Comandi o Errori

Usa il comando explain per ottenere una spiegazione dettagliata di un comando o errore:

copilot explain "comando o errore"

Esempio Pratico

copilot explain "permission denied while binding to port 80"

Copilot analizzerà l'errore e ti spiegherà la causa e possibili soluzioni.

3.3. Modalità Chat

Per una conversazione interattiva con Copilot, usa la modalità chat:

copilot chat

In questa modalità puoi porre domande in linguaggio naturale e ricevere assistenza contestuale continuativa.

3.4. Cambiare LLM

Puoi selezionare un modello linguistico diverso utilizzando:

/model

Questo comando ti permette di scegliere tra i modelli disponibili per ottimizzare le risposte in base alle tue esigenze.

Suggerimento

Sii quanto più specifico possibile nelle tue richieste. Più dettagli fornisci, migliori saranno i suggerimenti di Copilot.
